🧪 Dev Mode Test Page

📋 Test Instructions

This page helps you test the dev mode functionality. Follow these steps:

  1. Click the links below to test different scenarios
  2. Check if the dev mode button appears in the toolbar (wrench icon)
  3. Click the dev mode button to open the dev panel
  4. Test the connection toggle in the Connection tab
  5. Try evaluating expressions with Ctrl+Enter to see if gutter bars activate

🔗 Test Links

🔧 Dev Mode ON 📱 Normal Mode (dev mode off) 🐛 Dev Mode + Debug ON

✅ What to Look For

When Dev Mode is ON (?devmode=true):

  • ✅ Wrench icon appears in the toolbar (last row)
  • ✅ Clicking wrench opens dev mode panel
  • ✅ Panel has "Connection" and "Debug" tabs
  • ✅ Connection tab shows current status (red "Disconnected" initially)
  • ✅ "Connect" button enables mock connection
  • ✅ Status changes to green "Connected (Mock)"
  • ✅ "Disconnect" button disables mock connection
  • ✅ When "connected", expression evaluation should work

When Dev Mode is OFF (normal URL):

  • ✅ No wrench icon in toolbar
  • ✅ No dev mode panel available
  • ✅ Normal app functionality unchanged

🧬 Expression Testing

When dev mode connection is "ON", try evaluating these expressions in the editor:

(a1 (+ 1 2 3))

(a2 (map inc [1 2 3]))

(d1 (defn hello [] "world"))

(a1 (println "second a1 - this should be active"))

(d2 (range 10))

Expected behavior:

  • When you evaluate an expression with Ctrl+Enter, its gutter bars should become bright
  • Other expressions of the same type should become greyed out
  • Only the last evaluated expression of each type should be active

🎯 Success Criteria

The dev mode implementation is working correctly if:

  1. Dev mode elements only appear when ?devmode=true is in URL
  2. Connection toggle actually calls setConnectedToModule()
  3. Status display updates correctly
  4. Expression evaluation works when "connected" in dev mode
  5. Gutter highlighting activates/deactivates based on connection status
  6. No interference with normal app operation when dev mode is off

🔍 Debugging Tips

  • Open browser console to see debug messages
  • Look for "Dev mode active" and "Dev mode button added" messages
  • Check if isConnectedToModule() returns correct value in console
  • Use browser dev tools to inspect panel structure
