# Devlog — 2025-01-06

## Context
- Goal: decouple serial visualisation from firmware-only data flow and wire the built-in WASM interpreter into the editor evaluation pipeline.
- Scope touched: editor gutters, serial visualiser/controller, toolbar UI, settings UX, persistent configuration, WASM build helpers.

## Key Changes
### WASM Interpreter & Expression Handling
- Routed `aN/dN/sN` expression evaluation through the internal interpreter even when not targeting the hardware module (`structure.mjs`, `visualisationController.mjs`).
- Ensured no-op registrations are ignored; trimmed text is stored once per expression and resampled only when the definition changes.
- Added `refreshVisualisedExpression` path to retain past samples but recompute future predictions when the source expression edits.
- Synced gutter colour reports with visualiser state so traces reuse the pipeline palette.

### Serial Visualiser
- Rewrote sample management to maintain a symmetric window `[t-O, t+O]` with configurable offset and sample count.
- Draw loop now renders past values solidly, future values with alpha attenuation, and applies a static destination-out mask to create motionless “predictive gaps”.
- Added dynamic settings (offset, sample count, line width, mask toggle/opacities/stripe width) with UI wiring and persistence.
- Auto-opens the panel when a new expression is marked for visualisation and logs debug events for gutter button presses.

### UI & UX Enhancements
- Inserted a transport “bar progress” strip directly beneath the main toolbar row and drove it via WASM-evaluated `bar` values at ~30 FPS.
- Refined gutter marker layout to keep play/clear/visualise affordances clickable even on multi-line definitions; ensured icons adopt expression colours and respect accessibility attributes.
- Added console diagnostics to help track expression discovery and visualisation toggles.
- Moved the bar progress container out of the side toolbar stack and brightened its visuals for readability.

### Settings & Styling
- Centralised panel section background opacity in `base.css`, keeping light/dark themes readable while allowing targeted adjustments.
- Restored the settings panel theme and only increased background alpha to avoid washed-out text.
- Introduced disabled styling for mask-related sliders when the future mask toggle is off.
- Cleaned up reference panel layouts and panel toolbars (minor).

### Firmware & WASM Build Notes
- Added WASM-only storage for deferred outputs (`ModuLispInterpreter`) and wired evaluation helpers (`modulisp_api.cpp`), aligning with the JS side registration flow.
- Iterated on the build script/logging to surface access to time updates and serial output defaults.
- Confirmed firmware pushes logical time at the end of `tick()`; JS side calls `updateUseqWasmTime` and polls `bar` via the interpreter.
